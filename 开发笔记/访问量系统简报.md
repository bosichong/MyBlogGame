# 文章访问量系统简报

## 概述

访问量系统是游戏的核心数值系统之一，采用**修饰器模式（Modifier Pattern）**设计，通过多个独立的修饰器和事件来计算每篇文章的每日访问量。系统包含四大核心组件：**修饰器系统**、**事件系统**、**统计系统**、**文章统计系统**。

---

│   ├── BookSalesBoostModifier（加成）
│   ├── AnimationMovieModifier【已禁用】
│   ├── TaskBonusModifier（加成）

│

### 1.2 计算流程

```
每日计算 → 遍历每篇文章 → 依次应用修饰器 → 应用活跃事件 → 返回访问量
```

## 二、访问量生成算法

### 2.1 基础计算层（Priority 0-99）

#### SEO基础访问量（SEOBaseModifier）
- **优先级**：0
- **计算公式**：
  ```
  基础访问量 = (SEO值 / SEO满值) × 随机基础值
  随机基础值 = rand(文章等级基础值/3 ~ 文章等级基础值)
  ```
- **文章等级基础值**：
  | 等级 | 基础值 |
  |-----|--------|
  | 1   | 5      |
  | 2   | 12     |
  | 3   | 20     |
  | 4   | 40     |
  | 5   | 80     |

#### 文章质量加成（QualityModifier）
- **优先级**：10
- **计算公式**：
  ```
  访问量 = 基础访问量 × (1 + 质量分 / 150)
  ```
- **加成示例**：
  - 质量50：×1.33（+33%）
  - 质量100：×1.67（+67%）
  - 质量200：×2.33（+133%）

### 2.2 时间衰减层（Priority 100-199）

#### 时间衰减（TimeDecayModifier）
- **优先级**：100
- **有效年限**：半年（168天，游戏年336天的0.5倍）
- **衰减规则**：
  | 文章年龄 | 访问量处理 |
  |----------|-----------|
  | 0-14天   | 正常访问量 |
  | 14天-半年 | 30%概率获得1-5点随机访问量，随时间递减 |
  | 半年后   | 不再增加访问量 |

### 2.3 正向加成层（Priority 200-299）

#### 页面设计加成（DesignModifier）
- **优先级**：200
- **计算公式**：`加成 = 基础访问量 × 设计值 × 0.003`

#### 段位加成（RankModifier）
- **优先级**：210
- **计算公式**：`加成 = 基础访问量 × (段位² × 0.006)`
- **加成示例**：
  - 段位0：0%
  - 段位5：15%
  - 段位9：49%

#### RSS订阅加成（RSSModifier）
- **优先级**：220
- **有效期**：14天
- **计算公式**：
  ```
  加成 = RSS订阅数 × 0.1 × (1 - 天数/15)
  ```

#### 收藏加成（FavoritesModifier）
- **优先级**：230
- **有效期**：280天
- **计算公式**：`加成 = 收藏数 × 0.05 × (质量/100)`

#### 分享加成（ShareModifier）
- **优先级**：240
- **计算公式**：`加成 = 基础访问量 × (0.05 + min(访问量/50000, 0.2))`

#### 出书笔记加成（BookNotesModifier）
- **优先级**：240
- **计算公式**：`加成 = 基础访问量 × (综合评分/50)`
- **综合评分**：(文学 + 编程) / 2

#### 开源维护笔记加成（OpenSourceModifier）
- **优先级**：241
- **计算公式**：`加成 = 基础访问量 × (综合评分/50)`
- **综合评分**：三项技能平均值（文学+编程+绘画）

#### 动画影视加成（AnimationMovieModifier）
- **状态**：已禁用

#### 热门风向标加成（TrendModifier）
- **优先级**：250
- **计算公式**：`加成 = 基础访问量 × 0.2 × (质量/200)`
- **条件**：文章类型需匹配当前热门风向标

#### 连载加成（SerialBonusModifier）
- **优先级**：260
- **适用类型**：文学周刊、程序员周刊
- **计算公式**：`加成比例 = min(连载篇数/10 × 2%, 20%)`

#### 畅销书加成（BookSalesBoostModifier）
- **优先级**：270
- **有效期**：3年
- **加成曲线**：
  - 0-18月：线性上升 0% → 100%
  - 18-36月：线性下降 100% → 0%

#### 任务加成（TaskBonusModifier）
- **优先级**：500
- **任务类型加成**：
  | 任务类型 | 加成范围 |
  |----------|----------|
  | 第一篇博文 | 50%-100% |
  | 年度总结 | 50%-100% |
  | 节日文章 | 100%-400% |
  | 活动文章 | 200%-900% |
  | 专题文章 | 100%-300% |

### 2.4 负向限制层（Priority 300-399）

#### 主机流量限制（HostLimitModifier）
- **优先级**：300
- **逻辑**：检查当月访问量是否超过主机套餐上限

#### 欠费惩罚（SuspendPenaltyModifier）
- **优先级**：310
- **逻辑**：
  - 博客暂停：访问量=0
  - 恢复惩罚期间：访问量 × (1-惩罚比例)

#### 广告影响（AdImpactModifier）
- **优先级**：320
- **广告类型影响**：
  | 广告类型 | 访问量影响 |
  |----------|-----------|
  | 文字广告 | 0%         |
  | 图文广告 | -10%       |
  | 全站图文 | -15%       |
  | 弹窗广告 | -30%       |

## 三、事件系统

### 3.1 爆款文章事件（ViralEvent）
- **触发条件**：质量>150 且 社交能力>50，5%概率触发
- **持续时间**：3天
- **加成比例**：200%

### 3.2 平台推荐事件（FeaturedEvent）
- **触发条件**：SEO>150 且 设计值>150，3%概率触发
- **持续时间**：7天
- **加成比例**：50%

### 3.3 节日活动事件（ActivityEvent）
- **触发方式**：由外部系统调用触发
- **持续时间**：可配置
- **加成比例**：可配置

### 3.4 友链申请事件（FriendLinkPassiveEvent）
- **触发条件**：每月第1周第3天
- **效果**：从等级≤玩家等级-5的成员中随机选取1个被动申请

## 四、统计系统

### 4.1 数据结构（ViewsStatsData）
```gdscript
日统计: [{date, views, posts_count, sources}]
周统计: [{year, week, total_views, avg_views}]
月统计: [{year, month, total_views, avg_views}]
年统计: [{year, total_views, avg_views}]
来源统计: {direct, search, social, rss, favorite, event, task}
```

### 4.2 访问量来源分类
| 来源 | 说明 |
|------|------|
| direct | 直接访问（SEO基础+质量加成） |
| search | 搜索引擎 |
| social | 社交媒体 |
| rss | RSS订阅 |
| favorite | 收藏 |
| event | 事件加成 |
| task | 任务加成 |

### 4.3 统计指标
- 总访问量
- 今日访问量
- 最近7天/30天访问量
- 来源占比
- 增长率

## 五、付费文章处理

**规则**：付费文章（如小说连载）的访问量不计入博主总访问量，但正常参与单篇文章的访问量计算。
