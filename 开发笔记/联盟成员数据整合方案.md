# 联盟成员数据整合方案

## 核心思路：博客联盟为单一数据源

---

## 一、目标

**数据整合，保留所有现有功能。**

- 现有功能完整保留（主动申请、被动申请、维护日程等）
- 将 FriendLinkData.links 与 LeagueData.lm_members 合并为单一数据源
- 友链随机失效等功能暂时保持禁用状态
- 后续再逐步优化

---

## 二、现状问题

| 问题 | 说明 |
|------|------|
| 数据重复 | `FriendLinkData.links` 和 `lm_members.gd` 各存一份成员数据 |
| 需要转换 | FriendLinkManager 有 `_create_link_from_member()` 转换逻辑 |
| 维护困难 | 两套数据维护，容易不一致 |
| 多处引用 | GlobalDataManager、bloglm.gd、FriendLinkManager 各有一份成员列表 |

---

## 三、数据架构

### 3.1 当前架构

```
data/lm_members.gd (静态配置)
    ↓
GDManager.get_lm_members()  # GDManager 是 GlobalDataManager 的 autoload 别名
    ↓
    ├─→ bloglm.gd (lm_list 副本)
    └─→ FriendLinkManager (_league_members 副本)
            ↓
    FriendLinkData.links (友链数据，又复制一份)
```

**注意**：项目中 `GDManager` 是 `GlobalDataManager` 的 autoload 别名，所有代码应使用 `GDManager` 访问全局数据。

### 3.2 改造后架构

```
data/lm_members.gd (静态配置)
    ↓ 加载时扩展字段
LeagueData.lm_members (唯一运行时数据源)
    ↓
    ├─→ bloglm.gd (读取 lm_members)
    └─→ FriendLinkManager (直接读取，友链 = filter(is_friend_link))
            ↓
    FriendLinkData.pending_requests (仅待审核申请)
```

## 四、改造细节

### 4.1 联盟成员字段扩展

**静态配置** `data/lm_members.gd` - 不修改，保持原样

**运行时数据** `LeagueData.lm_members` - 每个成员字段：

```gdscript
{
  "id": 1,
  "blog_name": "星光博客",
  "blog_author": "星语者",
  "lv": 50,
  "type": 1,
  "quality": 50,
  "url": "starlight.com",
  "connectivity": true,       # 网站连通性（暂不启用）
  "is_friend_link": false,    # 是否已建立友链 ← 核心字段
  "add_date": "",             # 友链建立时间
  "last_check": ""            # 最后检查时间
}
```

### 4.2 LeagueData 新增属性和方法

```gdscript
# 新增属性
var lm_members: Array[Dictionary] = []

# 新增方法

## 初始化成员列表（从静态配置加载）
func init_lm_members():
    var static_members = GDManager.get_lm_members() if GDManager else []
    lm_members = []
    for m in static_members:
        lm_members.append({
            "id": m.get("id"),
            "blog_name": m.get("blog_name"),
            "blog_author": m.get("blog_author"),
            "lv": m.get("lv"),
            "type": m.get("type"),
            "quality": m.get("quality"),
            "url": m.get("url"),
            "connectivity": true,
            "is_friend_link": false,
            "add_date": "",
            "last_check": ""
        })

## 获取友链列表
func get_friend_links() -> Array[Dictionary]:
    return lm_members.filter(func(m): return m.get("is_friend_link", false))

## 获取单个成员
func get_member(id: int) -> Dictionary:
    for m in lm_members:
        if m.get("id") == id:
            return m
    return {}

## 添加/更新成员
func update_member(id: int, data: Dictionary):
    for i in range(lm_members.size()):
        if lm_members[i].get("id") == id:
            lm_members[i] = lm_members[i].merged(data)
            return
    # 新增
    lm_members.append(data)

## 标记友链
func mark_friend_link(id: int, is_link: bool):
    var member = get_member(id)
    if member.size() > 0:
        member["is_friend_link"] = is_link
        if is_link:
            member["add_date"] = Time.get_date_string_from_system()
        else:
            member["add_date"] = ""
```

### 4.3 FriendLinkData 简化

**删除：**
- `links` 数组

**保留：**
- `pending_requests`（待审核申请）

**新增：**
- `get_friend_links()` → 调用 `LeagueData.get_friend_links()`

**修改现有方法：**
- `has_link(member_id)` → 检查 `LeagueData.lm_members` 中 `is_friend_link`
- `get_active_links()` → 调用 `LeagueData.get_friend_links()`
- `remove_link(member_id)` → 设置 `lm_members[id].is_friend_link = false`

### 4.4 FriendLinkManager 改造

**删除：**
- `_league_members` 变量
- `set_league_members()` 方法
- `refresh_league_members()` 方法
- `_create_link_from_member()` 方法

**保留（适配新数据）：**
- `apply_for_link()` 方法（主动申请友链）
- `add_passive_requests()` 方法（被动申请友链，每月第1周第3天触发）
- `do_maintenance()` 方法（维护日程，审核申请）
- `process_pending_apply_result()` 方法（处理主动申请结果）
- `check_pending_requests()` 方法（检查到期申请）

**修改：**
- `get_links()` → `GDManager.get_league().get_friend_links()`
- `get_available_members()` → 从 `LeagueData.lm_members` 过滤
- `apply_for_link()` → 不再调用 `_data.add_link()`，改用 `LeagueData.mark_friend_link()`
- `do_maintenance()` → 审核通过时调用 `LeagueData.mark_friend_link()`
- `delete_link()` → 调用 `LeagueData.mark_friend_link(member_id, false)`
- `add_passive_requests()` → 从 `LeagueData.lm_members` 获取成员而非 `_league_members`

**新增：**
- `_get_league_data()` → 获取 LeagueData 引用

### 4.5 bloglm.gd 改造

**保留：**
- `donate_data`（捐赠数据，独立于成员列表）
- `quarter_activities()`（升级成员等级逻辑）
- 排序和查询方法（`sort_by_lv`、`sort_by_quality` 等）

**修改：**
- `lm_list` 改为从 `LeagueData.lm_members` 读取
- `join_lm()` → 调用 `LeagueData.update_member()`
- `exit_lm()` → 从 `LeagueData.lm_members` 移除玩家记录
- `_ready()` 中从 LeagueData 加载而不是 GlobalDataManager

**新增：**
- `_get_league_data()` → 获取 LeagueData 引用

### 4.6 存档改造

**SaveManager 修改：**

```gdscript
func serialize_league_data(data: LeagueData) -> Dictionary:
    return {
        "is_joined": data.is_joined,
        "donation_total": data.donation_total,
        "donation_history": data.donation_history,
        "rank_overall": data.rank_overall,
        "rank_literature": data.rank_literature,
        "rank_tech": data.rank_tech,
        "rank_art": data.rank_art,
        "donation_ranking": data.donation_ranking,
        "lm_members": data.lm_members  # ← 新增
    }

func deserialize_league_data(data: LeagueData, dict: Dictionary):
    # ... 保留现有字段 ...
    data.lm_members = dict.get("lm_members", [])
```

**FriendLinkData 存档：**

```gdscript
func serialize_friendlink_data(data: FriendLinkData) -> Dictionary:
    return {
        # 删除 "links" 字段
        "pending_requests": data.pending_requests,
        "last_maintenance_date": data.last_maintenance_date,
        "auto_settings": data.auto_settings,
        "is_league_member": data.is_league_member,
    }
```

**存档迁移：**
- 反序列化时检查旧存档中的 `links` 字段
- 将每个 link 转换为 `lm_members` 中的 `is_friend_link` 状态

## 七、注意事项

### 7.1 初始化时机

- LeagueData.init_lm_members() 应在游戏启动或读取存档后调用
- 如果读取存档时有 lm_members，直接使用；否则调用 init_lm_members()

### 7.2 数据一致性

- 所有修改联盟成员的地方都通过 LeagueData
- 不要直接修改 GlobalDataManager.loaded_data 中的数据

### 7.3 兼容性

- 旧存档需要迁移数据（links → is_friend_link）
- 迁移逻辑在 deserialize_league_data 中实现
