# 属性触发任务

## 任务定位

**基于属性阈值触发，以模块背景为依托**

不是纯随机事件，而是当某属性低于/高于阈值时触发应对任务。

---

## 触发机制

```
属性值检测 → 低于阈值 → 触发对应任务 → 玩家应对 → 属性恢复
```

---

## 模块与属性对应表

| 模块 | 监控属性 | 阈值条件 | 触发任务背景 |
|------|----------|----------|--------------|
| 安全防护 | 安全值 | < 20 | 被黑、DDoS、漏洞攻击 |
| 网站维护 | SEO值 | < 30 | 算法惩罚、排名下跌 |
| 博客联盟 | 活跃度 | < 10 | 被警告、友链失效 |
| 广告联盟 | 审核值 | < 50 | 账户封禁、审核失败 |
| 域名主机 | 到期天数 | ≤ 7 | 域名/服务器即将到期 |
| 投资理财 | 存款到期 | ≤ 0 | 定期存款到期提醒 |

## 设计要点

1. **阈值检测**：游戏内定时检测属性值，低于阈值触发任务
2. **任务应对**：玩家需要操作完成应对（如清理友链、续费域名）
3. **属性恢复**：完成任务后属性值恢复，避免重复触发
4. **模块背景**：每个任务的剧情与对应模块背景一致，增强沉浸感
