# 开场动画构建文档（s003 场景）

> 本文档定义了游戏开场动画的设计方案，包括对白文案、场景构建和动画剧本。

---

## 一、设计理念

### 1.1 时间定位

**2001年12月，冬夜。**

这是博客进入中国的前夜——距离2002年8月"博客中国"成立还有大半年。此时的互联网属于：

- **BBS 论坛**：水木清华、天涯社区、榕树下
- **个人主页**：网易 Geocities、China.com 个人空间
- **即时通讯**：QQ 刚刚兴起（2000年上线）
- **拨号上网与宽带萌芽**：ISDN 开始普及，但大多数人是拨号

### 1.2 主角人设

- **身份**：一个热爱写作、有表达欲望的年轻人
- **状态**：对现有互联网表达方式感到不满，渴望有一个真正属于自己的角落
- **痛点**：BBS 帖子太碎片，论坛讨论太口水，没有人认真写长文

### 1.3 核心主题

**"表达欲与互联网的第一次碰撞"**

## 二、对白文案（重写版）

### 2.1 整体基调

- 怀旧感：90年代末到2000年代初的互联网氛围
- 孤独感：一个人面对屏幕，深夜上网
- 渴望感：渴望被看见，渴望认真表达，渴望一个角落

### 2.3 文案修改说明

| 原版问题 | 新版改进 |
|----------|----------|
| "宽带是快了，但内容好像变慢了" —— 过于技术化 | 改为更感性、更生活化的场景描写 |
| "BBS热帖太零散太快餐" —— 直接吐槽，缺乏画面 | 改为具体的画面：短的像烟，长的像废话 |
| "缺少真正属于自己的角落" —— 过于抽象 | 改为"这个互联网很大，却没有我的位置" |
| 整体节奏：原版叙事较平 | 新版增加了"邮件转折"，形成叙事高潮 |

## 三、场景构建

### 3.1 整体布局

```
┌─────────────────────────────────────────────────┐
│                    窗外夜空                       │
│  （半透明，表现冬夜氛围）                         │
├─────────────────────────────────────────────────┤
│                                                 │
│            ┌─────────────────┐                 │
│            │  CRT显示器      │                 │
│            │  (主体画面)      │                 │
│            │  论坛页面        │                 │
│            │  邮件客户端      │                 │
│            └─────────────────┘                 │
│                                                 │
│         💻（台灯暖光）    ⌨️（键盘）             │
│                                                 │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │
│              （书桌/地毯）                       │
└─────────────────────────────────────────────────┘
```

### 3.2 场景元素详解

| 元素 | 视觉特征 | 动画要点 |
|------|----------|----------|
| **CRT显示器** | 厚重边框，屏幕微凸，有扫描线效果 | 开机闪烁，白天晚上模式切换 |
| **论坛页面** | 典型2001年BBS风格，绿色/蓝色链接 | 滚动刷新，帖子层层堆叠 |
| **窗外夜空** | 暗蓝色，几颗星星，远处楼灯光 | 微弱的闪烁，表现冬夜寂静 |
| **台灯** | 暖黄色光晕，照亮桌面一角 | 持续点亮，有轻微呼吸感 |
| **键盘** | 老式键冒，略有磨损 | 敲击时有轻微震动效果 |
| **鼠标** | 机械滚轮鼠标，有线 | 移动，翻页点击 |
| **邮件客户端** | 类似Outlook Express风格 | 收件箱图标，闪烁提示 |

### 3.3 色彩方案

| 状态 | 主色调 | 辅色 | 说明 |
|------|--------|------|------|
| **前段（1-27s）** | 深蓝灰 #1a1a2e | 屏幕蓝光 #4a90d9 | 孤独感，冷调 |
| **转折（27-33s）** | 黑色 #000000 | 无 | 静默，悬念 |
| **后段（33-45s）** | 暖黄 #f4a460 | 白色 #ffffff | 希望感，温暖 |

## 五、音频设计

## 六、技术实现要点

### 6.1 Godot 场景结构建议

```
s_003 (Node2D - 主场景)
├── 背景层 (Background)
│   ├── 窗外夜空 (Sprite2D/ParallaxLayer)
│   └── 室内家具 (Sprite2D)
├── 显示器层 (Monitor)
│   ├── 屏幕框架 (Sprite2D)
│   ├── 论坛页面 (Control/RichTextLabel)
│   ├── 邮件界面 (Control)
│   └── 扫描线效果 (Polygon2D/Shader)
├── 交互层 (Interaction)
│   ├── 键盘 (Sprite2D)
│   ├── 鼠标 (Sprite2D)
│   └── 手部剪影 (AnimatedSprite2D)
├── UI层 (UI)
│   ├── 配文显示 (RichTextLabel - 使用visible_ratio)
│   └── 跳过按钮 (Button)
└── 音频层 (Audio)
    ├── 环境音效 (AudioStreamPlayer)
    └── 邮件提示音 (AudioStreamPlayer)
```

### 6.2 动画实现方案

**方案A：纯 AnimationPlayer（当前方案）**

- 优点：可视化编辑，直观
- 缺点：复杂的时间轴难以维护

**方案B：代码驱动 + Timeline**

```gdscript
# 推荐：在 s_003.gd 中使用时间轴控制
extends Node2D

var timeline := [
    {"time": 0.0, "action": "fade_in", "duration": 3.0},
    {"time": 3.0, "action": "show_text", "text": "窗外是北风，家里是暖的。"},
    # ... 更多时间点
]

func _process(delta):
    # 时间轴逻辑
    pass
```

### 6.3 文字显示优化

当前使用 `visible_ratio` 实现打字机效果，建议：

1. **独立控制**：每个 Label 的显示时机由代码控制，而非硬编码在动画中
2. **字体回退**：确保 OPPOSans-M.ttf 不可用时有系统字体回退
3. **响应式布局**：考虑不同分辨率下的文字位置
