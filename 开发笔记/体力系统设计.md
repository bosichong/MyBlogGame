# 体力系统设计

**日期**：2026-05-16

---

## 核心理念

体力每天固定重置为满，不累积。

体力是"过滤器"：低级内容消耗少，但访问量也少；高级内容消耗多，但访问量高。玩家自然选择高级内容。

---

## 体力上限（按等级）

| 等级 | 上限 |
|------|------|
| 1-10级 | 30-40 |
| 11-30级 | 50-70 |
| 31-50级 | 75-85 |
| 51-80级 | 85-100 |
| 81级+ | **100** |

---

## 体力消耗

| 活动 | 体力 |
|------|------|
| 学习技能 | 15-30 |
| 低级博客 | 20 |
| 中级博客 | 30-40 |
| 高级博客 | 50-60 |
| 顶级（写书/出版） | 70-80 |
| 网站维护 | 15-20 |

---

## 文章访问量

| 等级 | 体力 | 访问量 |
|------|------|--------|
| 入门级 | 20 | 50-100 |
| 初级 | 30 | 150-300 |
| 中级 | 40 | 400-700 |
| 高级 | 50-60 | 800-1500 |
| 顶级 | 70-80 | 2000-5000 |

---

## 设计逻辑

1. 体力每天重置为满
2. 体力用于三选一：学技能 / 写文章 / 做维护
3. 低级文章访问量差，发多篇不如1篇高级
4. 不学习技能就解锁不了高级内容，访问量上不去
5. 升级后体力上限增加

---

## 核心循环

```
体力（每天重置）
  ↓
├── 学技能(15-30) ─→ 解锁高级内容 ─→ 更高访问量
├── 写文章(20-80) ─→ 直接获得访问量
└── 做维护(15-20) ─→ 维护SEO值

升级 ─→ 体力上限增加(30→100)
```
