# 性能优化待办

## 背景

4 倍速运行一段时间后卡顿。根因：每天的计算量随游戏时长和文章数增长，但 4x 留给每帧的时间窗口只有 100ms。

---

## 已完成优化

### ✅ SerialBonusModifier 缓存连载篇数
- 消除 O(n²)，每日遍历 posts 只缓存一次，而非每篇文章都遍历全部 posts
- 改动：`scripts/views/modifiers/SerialBonusModifier.gd` + `scripts/views/ViewsCalculator.gd`

### ✅ 付费循环合并 4→1
- 合并收集、算质量分、算结算为一次遍历
- 改动：`scripts/blogger.gd`

### ✅ BookSalesBoost 脚本实例缓存
- 原每次 apply() 都 `load().new()`，改为懒加载缓存
- 改动：`scripts/views/modifiers/BookSalesBoostModifier.gd`

### ✅ 评论 ID 生成消除全表扫描
- `_generate_comment_id()` 从遍历全部评论找 max_id，改为时间戳 + 随机数
- 改动：`scripts/managers/CommentManager.gd`

### ✅ SpamPenaltyModifier 垃圾评论缓存
- 原每篇文章 `get_comments(post_id).filter()`，改为 `refresh_post_cache()` 每日一次 + O(1) 查表
- 改动：`scripts/views/modifiers/SpamPenaltyModifier.gd` + `scripts/managers/CommentManager.gd`

### ✅ TaskManager trigger_type 索引
- 原 `check_tasks_by_trigger()` 每日遍历全部 ~140 个任务，改为按 trigger 预索引
- 改动：`scripts/task_system/TaskManager.gd`

### ✅ ViewsCalculator 老文章 continue 跳过
- 在 `calculate_daily()` loop 入口检查文章年龄，>84 天直接 continue
- 跳过 18 个修饰器 + 事件 + 统计记录
- 改动：`scripts/views/ViewsCalculator.gd`

### ✅ 月末归档（物理分离老文章）
- `BloggerData` 新增 `archived_posts` 字段，月末将 >84 天的文章移入
- `calculate_daily()` / `check_all_articles()` / 收藏增长 → 只扫 `posts`
- 付费收入 / 任务计数 / 收藏衰减 / UI → 双数组求和
- `sync_to_post()` / `SerialBonusModifier` → 双数组扫描
- 旧存档兼容（`archived_posts` 默认为空数组）
- 改动：10 个文件（BloggerData/SaveManager/blogger/CommentManager/TaskManager/dashboard/YearlySummary/main/SerialBonusModifier）

### ✅ `sync_all_posts()` 冗余重建（内联到 `check_all_articles()`）
- `check_all_articles()` 生成评论后直接 `post["comments"] += generated`，O(1) 同步
- 移除了 `blogger.gd` 中每日路径的 `sync_all_posts()` 调用（函数保留供非日常路径使用）
- 改动：`scripts/managers/CommentManager.gd` + `scripts/blogger.gd`

### ✅ `get_comment_count()` 全量 filter → O(1) 缓存
- `_post_comment_counts` 字典在 `refresh_post_cache()` 中一次构建
- `check_article_comments()` 初始计数改为 `get_comment_count_cached(post_id)` 查表
- 消除 168 篇文章 × 5000 条评论 = 840K 次 filter 回调
- 改动：`scripts/managers/CommentManager.gd`

## 🟡 待优化 — 中优先级

### [ ] P1: `Blogger.blog_data` getter 低效

- 位置：`scripts/blogger.gd:188-218`
- 问题：每次 getter 重建 15 字段 Dictionary + `get_node_or_null("/root/Tongji")` 场景树遍历
- `update_ui()` 每天调用 1 次（4x）× 内部多次访问 `Blogger.blog_data.xxx`
- 方案：减少 blog_data 访问次数；在 main.gd 中一次取数据分多控件赋值

### [ ] P1: `select_league_member()` 每次 copy lm_members

- 位置：`scripts/managers/CommentManager.gd:304-314`
- 问题：每次生成评论调用 `GDManager.get_lm_members()` → `lm_list.duplicate()` 复制整个数组
- 但 lm_members 是只读配置，不需要每次都 copy
- 方案：去掉 `duplicate()`，或缓存一次

---

## 🟢 待优化 — 低优先级

### [ ] P2: 收藏更新 loop 的二次查询

- 位置：`scripts/blogger.gd:753-767`
- 问题：遍历所有文章调用 `views_calculator.get_post_stats(post_id)` 查询今日访问量
- `calculate_daily()` 已经算过每篇文章的今日访问量，但没有返回
- 方案：`calculate_daily()` 返回 per-post 数据，收藏更新直接读取

### [ ] P2: 统计数组无限增长

- 位置：`scripts/views/PostStatsManager.gd` + `scripts/views/ViewsStatsManager.gd`
- 问题：`daily_views` 和 `daily_stats` 每天追加，只增不减
- 游戏 20 年后每篇文章 6720 条记录
- 方案：存档时裁剪到最近 365 天
