# 友链系统简报

## 概述

友链 = 博客联盟成员之间的深度合作。只能从联盟成员中选择。

---

## 已完成功能

**数据层**
- FriendLinkData：友链数据 + 待处理申请 + 自动设置
- FriendLinkManager：完整管理器
- 自动存档 / 读取

**加成效果**
- 流量：等级越高加成越高（lv40+ +100/日）
- SEO：友链数量贡献上限 20 点

**申请方式**
- 主动申请：管理面板操作，消耗体力，等待结果
- 被动申请：每月第1周第3天，系统从等级≤玩家等级-5的成员中随机选取1个发起申请

**审核处理**
- 维护日程：根据设置自动通过/拒绝
- 等级判断：对方等级 > 玩家等级 → 直接通过；对方等级 <= 玩家等级 → 等级差过小则拒绝

**UI 面板**
- 查看友链列表、状态、等级、加成
- 手动删除友链
- 申请友链（从联盟成员选择）
- 自动处理设置（等级差限制、失效/垃圾友链处理）
- 待处理申请列表（区分主动/被动申请）

**事件系统**
- 独立事件类 FriendLinkPassiveEvent，耦合低
- 事件系统触发 → 创建被动申请 → 友链系统存储 → 日程系统审核

---

## 申请流程

```
主动申请（玩家操作）
  管理面板选成员 → 消耗体力 → 加入待审列表 → 等待 → 按成功率判定结果

被动申请（系统事件）
  每月第1周第3天 → 事件触发 → 筛选等级≤玩家等级-5的成员 → 随机选取1个加入待审列表 → 维护日程审核
```
