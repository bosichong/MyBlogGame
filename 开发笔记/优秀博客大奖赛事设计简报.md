# 优秀博客大奖赛事设计（v3 — 联盟排名驱动版）

## 判定规则

| 联盟排名 | 获奖结果 | 称号 |
|---------|---------|------|
| `排名 < 1`（未加入或查无此人） | 无奖项 | 无 |
| `1 ≤ 排名 ≤ 10` | 优秀博客大奖 | 博客之星 |
| `11 ≤ 排名 ≤ 25` | 入围 | 博客新锐 |
| `26 ≤ 排名 ≤ 50` | 提名 | 博客新星 |
| `排名 ≥ 51` | 无奖项 | 无 |
| 连续同结果 | 蝉联触发 | 追加蝉联提示 |

## 排名来源

- 排名取自 `Lm.jhph`（联盟江湖榜，按等级降序排列）
- 玩家 id = 888，在 `Lm.jhph` 中的索引（从 1 开始）即为排名
- 若玩家未加入联盟或 `Lm.jhph` 为空，结果为无奖项

## 蝉联判定

- 每次揭晓时记录本次结果等级（none/nominate/shortlist/win），与上一次对比，相同且非 NONE 则触发蝉联
- 蝉联提示：
  - 提名蝉联：再次获得提名，继续精进，未来可期！
  - 入围蝉联：再次入围，实力稳健，距大奖仅一步之遥！
  - 获奖蝉联：蝉联优秀博客大奖！传奇仍在继续！

## 场景剧本（v3 — 细化版）

### 通用设计原则

- **像素画风**：所有场景复用现有 `assets/scens/s003/` 下的邮件素材（email.png、email2.png、email3.png），像素美术风格一致
- **复用现有场景结构**：通知/未获奖揭晓复用现有的 `tongzhi.tscn` 场景结构（`ColorRect` 遮罩 → `email_ico` 闪烁动画 → `emailui` 淡入）；评选复用 `pingxuan.tscn` 结构（`RichTextLabel` 5 条 + `AnimationPlayer` 控制 `visible_ratio` 逐条显示）
- **色调**：通知阶段黑底半透明遮罩（`Color(0,0,0,0.67)`），邮件界面蓝白配色；评选阶段深色文字浮于黑底；揭晓获奖用暖色（金色/红色）突显仪式感
- **操作方式**：三个阶段均通过按钮/动画完成触发 → 确认 → 返回流程，玩家无需复杂操作

### 评选阶段

#### 场景设计

```
场景设定：评审会议室 / 深夜书房 · 静态背景
背景构成：
  主背景：深色木质书桌或会议桌的像素静物画
          桌面散落：打印出的博客文章纸堆、咖啡杯、老式台灯、钢笔
          桌角放着一叠信封（标注"提名"字样）
  氛围：暖色台灯光晕照亮桌面中央区域，四周渐暗
        整体色调偏暖棕（#2a1f14 渐变为 #0d0a07 边缘）
  
  动态元素（可选）：
  ① 台灯灯芯微弱呼吸动画（极缓亮度变化，周期 3s）
  ② 纸堆上方飘浮极淡的"尘埃粒子"（缓慢上下浮动，低透明度）
  ③ 钟表秒针声（极低音量，每 3s 一声）

  镜头：固定中景，以桌面中央纸堆为中心
```

> **为什么是这个场景？** 评选阶段没有激烈的事件发生，评审团在幕后审阅博客。用"深夜里评审桌上的纸堆"作为视觉锚点，能传递"认真、审慎、有分量"的感觉。同时黑底 + 纸堆剪影的实现成本极低（1 张像素背景图 + 几个粒子节点），与现有美术风格一致。

#### 场景结构

| 节点 | 类型 | 用途 |
|------|------|------|
| `bg` | `Sprite2D` | 桌面纸堆背景图 |
| `bg/灯` | `Sprite2D` | 台灯剪影（可选，可合并在 bg 中） |
| `bg/粒子` | `GPUParticles2D` | 微尘浮动粒子（可选） |
| `01` ~ `05` | `RichTextLabel` | 5 条评选字幕，`visible_ratio` 初始为 0 |
| `AnimationPlayer` | `AnimationPlayer` | 控制各字幕的 `visible_ratio` 动画（逐条显示后自动返回） |

字幕以文字从左侧依次"划入"（visible_ratio 0→1 动画）的方式显示，每条显示约 2.5s 后淡出，下一条紧跟。

#### 动画节奏

| 时序 | 画面 | 音效 |
|------|------|------|
| t=0~1.0 | 黑屏渐亮 → 桌面纸堆背景淡入（1.0s），台灯暖光晕 | 秒针走时声（轻柔）起 |
| t=1.0~3.5 | 字幕 01 从左侧划入（`visible_ratio` 0→1），停留至 t=3.5 | — |
| t=3.5~4.0 | 字幕 01 划出（`visible_ratio` 1→0） | — |
| t=4.0~6.5 | 字幕 02 划入→停留→划出 | — |
| t=7.0~11.2 | 字幕 03 划入→停留→划出 | — |
| t=11.6~17.0 | 字幕 04 划入→停留→划出（2010 届于此期间同时显示「入围名单」RichTextLabel） | — |
| t=17.5~21.2 | 字幕 05 划入→停留→划出 | — |
| t=22.0~23.0 | 背景渐暗至黑屏（1.0s） | 秒针声渐弱，结束 |
| t=23.0 | 动画结束，`_on_animation_player_animation_finished` 触发 → 自动返回主场景 | — |

总长度 23s，自动播放，无需玩家操作。

#### 场景 A：颁奖典礼舞台（排名 1~10）

```
场景设定：颁奖典礼现场 · 夜晚
画面构成（自上而下）：
  ① 顶部幕布装饰（暗红色像素幕布，上部悬挂横幅挂轴）
  ② 舞台背景 = 一幅巨大的获奖海报（横长形，覆盖舞台后方大部分区域）
     海报风格：红底金字，传统"荣誉红榜"样式
               顶部大字"第 N 届中文优秀博客大奖"
               中部列出 6 位获奖者（玩家名在第 6 行，金色高亮）
               底部留白给评语/称号
  ③ 舞台地板（木质纹理，暖色灯光照射）
  ④ 舞台中央偏左/右的位置放置立式话筒（像素金色造型）
  ⑤ 画面最前方：观众席剪影（稀疏，仅第一排轮廓）

镜头：固定全景，以舞台和背景大海报为中心，无需镜头运动。
       大海报本身就占画面 50% 以上面积，是所有视觉的焦点。
```

> **为何是"背景大海报"而非大屏幕？** 2005~2025 年间，博客颁奖多为小型会场，不具有大型 LED 屏幕的条件。正面悬挂一幅巨大的实体印刷海报（类似红榜/荣誉榜），既是时代限制的体现，也更有传统仪式感。从实现角度看，一张海报素材 + 几组文字标签即可完成，无需复杂动画系统。

#### 场景 B：邮箱结果通知（排名 ≥ 11）

**无奖项（排名 0 或 ≥ 51）/ 提名（排名 26~50）/ 入围（排名 11~25）共用同一场景框架**，场景结构完全复用**通知阶段**的邮箱界面（`tongzhi.tscn` 结构），仅在邮件正文内容上区分。

#### 场景说明

复用同一套 `ColorRect + email_ico + emailui` 结构，动画流程也与通知阶段一致：邮件图标闪烁 → 邮箱界面淡入 → 玩家阅读 → 退出返回。区别在于：

- 邮件标题和正文内容改为揭晓结果
- 不需要任何场景复杂度上的差异——提名/入围/无奖项在视觉上保持统一，仅通过措辞区分等级

#### 邮箱正文模板

**无奖项（排名 0 或 ≥ 51）：**

> **标题**：第 N 届「中文优秀博客大奖」评选结果
>
> **正文**：
> 尊敬的博主，
>
> 第 N 届「中文优秀博客大奖」获奖名单已正式公布。
> 很遗憾，您的博客「博客名」未获得本届任何奖项。
>
> 本届共收到 X 份参赛作品，组委会感谢您的参与。
>
> 下一届评选将在 {Y} 年后举行，期待您的成长与蜕变。
>
> 评语：「坚持写作，时间会给你答案。」
>
> 中文优秀博客大奖评委会

**提名（排名 26~50）：**

> **标题**：恭喜！您的博客获得第 N 届优秀博客大奖提名
>
> **正文**：
> 尊敬的博主，
>
> 恭喜您的博客「博客名」获得第 N 届「中文优秀博客大奖」提名！
>
> 本届共有 50 位优秀博主获得提名，您的博客名列其中。
> 您从众多参赛者中脱颖而出，这是对您坚持创作的肯定。
>
> 您获得称号「博客新星（年份）」！
> [蝉联]再次获得提名，继续精进，未来可期！
>
> 评语：「每一篇认真的文字都值得被看见。」
>
> 中文优秀博客大奖评委会

**入围（排名 11~25）：**

> **标题**：恭喜！您的博客入围第 N 届优秀博客大奖
>
> **正文**：
> 尊敬的博主，
>
> 恭喜您的博客「博客名」入围第 N 届「中文优秀博客大奖」！
>
> 您从本届数百位提名者中脱颖而出，成为 20 位入围者之一。
> 入围即是对您博客品质的高度认可。
>
> 您获得称号「博客新锐（年份）」！
> [蝉联]再次入围，实力稳健，距大奖仅一步之遥！
>
> 评语：「您的博客代表了中文独立博客的中坚力量。」
>
> 中文优秀博客大奖评委会

> **注意**：排名 ≥ 11 的所有结果都不出现现场典礼场景，统一保持"邮件通知"的私密阅读氛围。
> 提名/入围与无奖项的区别体现在正文措辞和称号授予上，而非场景复杂度上。这强化了"获奖才有仪式感"的设定。

## 数据流

```
通知时间到 → 显示中立邀请弹窗 → 确认 → 邮箱动画（无预判）
评选时间到 → 显示中立评选弹窗 → 确认 → 字幕叙事
揭晓时间到 → 显示结果弹窗 → 确认
                ↓
          读取 Lm.jhph 联盟排名
                ↓
          判定排名区间 → 结果等级
                ↓
          对比 award_history → 蝉联
                ↓
          动态生成文案 → 现场/邮件场景
                ↓
          记录 award_history + 标记里程碑
```
