# 评论系统

## 概述

评论来自博客联盟成员。评论生成后正面直接显示，负面评论需要通过「评论管理」日程处理。

***

## 完整流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    每日结算 (update_blog_views)             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 访问量达标                                               │
│     └─ 每500访问 = 1次生成机会                               │
│                                                             │
│  2. 概率判定                                                │
│     └─ 概率 = min(质量 / 200, 50%)                         │
│                                                             │
│  3. 概率通过 → 随机选择联盟成员                              │
│                                                             │
│  4. 评论类型判定                                             │
│     ├─ 90% → 正面评论（正常显示）                           │
│     └─ 10% → 负面评论（标记为待处理，正常显示）              │
│                                                             │
│  5. 生成评论                                                │
│     └─ 负面评论: status=spam, is_spam=true                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
                                                              │
                                                              ▼
┌─────────────────────────────────────────────────────────────┐
│              玩家执行「评论管理」日程 (do_maintenance)         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 处理待管理评论                                           │
│     └─ is_spam=true → status=hidden                        │
│     └─ SEO +5（每条+5）                                     │
│                                                             │
│  2. 特殊事件处理                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 垃圾评论机制

| 项目 | 说明 |
|------|------|
| **生成概率** | 每次生成机会中，10%概率生成负面评论 |
| **生成来源** | 随机联盟成员 |
| **初始状态** | status=spam, is_spam=true |
| **显示** | 正常显示在"全部评论"中，标记"⚠️待处理" |
| **处理方式** | 「评论管理」日程自动隐藏 |
| **奖励** | 处理后 SEO+5（每条） |

### 负面评论模板

由联盟成员生成负面评论，内容包括：
- 批评文章质量
- 质疑技术观点
- 表达不满情绪

## 触发机制

### 阶段一：评论生成（每日自动）

**触发时机**：`daily_activities()` 中的 `update_blog_views()` 末尾自动执行

```gdscript
# blogger.gd - 每日结算时自动触发
func update_blog_views():
    # ... 访问量计算 ...
    
    # 生成评论（自动，无需玩家操作）
    var comment_manager = GDManager.get_comment_manager()
    if comment_manager:
        comment_manager.check_all_articles()
```

**生成条件**：
- 每500访问 = 1次生成机会
- 生成概率 = min(质量 / 200, 50%)
- 评论类型：90%正面/10%负面
- 联盟评论：随机抽取联盟成员生成
- 友链评论：🔸 **暂不实现，后续单独研究**

**生成后状态**：hidden（隐藏）

### 阶段二：评论显示（需要日程）

**触发方式**：玩家在日程中选择「评论管理」，消耗10点体力

```gdscript
func do_maintenance():
    # 显示隐藏评论（hidden → normal）
    reveal_hidden_comments()
    
    # 删除垃圾评论（spam → 删除）
    auto_delete_spam()
    
    # 特殊事件处理
    process_special_events()
```

## 评论数量

基础机会：每500访问 = 1次生成机会

| 质量分 | 生成概率 | 500访问期望 |
| --- | ---- | ------- |
| 0   | 0%   | 0条      |
| 50  | 40%  | 0.4条    |
| 80  | 64%  | 0.64条   |
| 95  | 76%  | 0.76条   |
| 100 | 80%  | 0.8条    |

> 实际结果受概率影响，质量越高越容易生成评论

***

## 联盟成员抽取

当需要生成联盟评论时，从符合以下条件的联盟成员中随机抽取1人：

- 博客类型与文章类型匹配（综合/文学/编程）

> 注：已取消等级限制

***

## 评论内容

**联盟成员**：根据文章类型生成自然评论，使用占位符替换

```gdscript
# 评论模板结构
{
    "技术类": "这篇文章对 {topic} 的讲解很透彻，受益匪浅",
    "教程类": "跟着步骤操作成功，感谢分享",
    "文学类": "...",
    "综合类": "...",
}
```

生成时根据文章内容替换 `{topic}` 等占位符为具体内容。

**友链博主**：常规互动、任务祝贺、互链深化、紧急提醒

**特殊任务**：在任务中单独创建评论内容

## 评论状态

| 状态          | 说明              | 生命周期              |
| ----------- | --------------- | ----------------- |
| **hidden**  | 隐藏状态，生成后默认状态    | 生成 → 待审核 → normal |
| **normal**  | 正常显示的评论         | 最终状态              |
| **pending** | （已废弃，合并到hidden） | -                 |
| **spam**    | 垃圾评论，自动删除后SEO+5 | 待删除               |

> 注意：新流程中评论生成后为 hidden，执行「评论管理」日程后变为 normal

***

## 游戏影响

流量加成计算：**每篇文章**的评论数决定加成

| 单篇文章评论数 | 流量加成    |
| ------- | ------- |
| ≥10     | +50/日   |
| ≥50     | +500/日  |
| ≥100    | +1000/日 |

> 注意：是单篇文章的评论数，而非总评论数
